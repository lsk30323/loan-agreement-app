# 차용증(금전소비대차) 관리 앱

채권자·채무자·이자·상환조건을 담은 **표준 차용증을 작성·출력**하고, 실제 상환 내역을 입력하면 **남은 잔액과 앞으로 매월 갚아야 할 금액을 자동으로 갱신**해 주는 로컬 우선(local-first) 정적 웹앱입니다.

> 사양서(전체 명세): [`loan-agreement-app-spec.md`](./loan-agreement-app-spec.md)

## 앱 목표 (3가지)

1. **차용증 작성·출력** — 표준 항목(채권자/채무자/금액/이자/상환일/상환기간/상환방법/지급방법/작성일)을 입력하면 인쇄·PDF 저장 가능한 차용증 문서를 생성합니다.
2. **상환 원장 + 실시간 갱신** — 실제 상환 내역을 입력하면 현재 잔액을 계산하고, **앞으로의 권장 월 상환액 / 예상 완납일**을 다시 계산해 보여줍니다(핵심 기능).
3. **범용성·정확성** — 0~1,000억 원, 4가지 상환방식(만기일시·원금균등·원리금균등·자유상환), 이자제한법 연 20% 상한 검증, 한글 금액 표기까지 정확히 처리합니다.

## 실행 방법 (빌드 단계 없음)

순수 HTML + CSS + Vanilla JS(ES Modules)로, **빌드/번들러/외부 의존성이 없습니다.**

ES module은 `file://`로 직접 열면 CORS 정책 때문에 로드되지 않으므로 **반드시 정적 서버를 경유**해야 합니다.

```bash
# 1) 정적 서버 실행 (둘 중 하나)
python -m http.server 8000
#   또는
npm run serve            # = python -m http.server 8000

# 2) 브라우저에서 접속
#   http://localhost:8000
```

## 휴대폰에 설치 (PWA)

이 앱은 **설치형 PWA**입니다 — 휴대폰 홈 화면에 일반 앱처럼 설치되고, 설치 후에는 **오프라인에서도** 열립니다(앱 셸을 서비스워커가 캐시).

> 설치하려면 **HTTPS 주소** 또는 `localhost`로 접속해야 합니다(서비스워커 요건). 파일을 인터넷에 올리는 방법은 아래 [배포](#배포) 참고.

- **Android (Chrome)**: 사이트 접속 → 메뉴(⋮) → **‘앱 설치’** 또는 ‘홈 화면에 추가’.
- **iPhone (Safari)**: 사이트 접속 → 공유 버튼 → **‘홈 화면에 추가’**.

설치 후에도 데이터는 그 **기기의 브라우저 저장소(localStorage)** 에만 있습니다. PC·휴대폰 간 이동은 **내보내기/가져오기(JSON)** 버튼을 사용하세요.

## 배포

서버·빌드가 없는 정적 파일이라 **어떤 정적 호스팅에도 그대로** 올릴 수 있습니다. PWA 설치를 위해 HTTPS를 제공하는 곳을 권장합니다.

| 방법 | 절차 |
|---|---|
| **Netlify Drop** (가장 빠름) | <https://app.netlify.com/drop> 에 이 폴더를 드래그&드롭 → 즉시 https 주소 발급 |
| **Cloudflare Pages / Vercel** | 무료 계정 + 폴더 업로드(또는 git 연결) → https 자동 |
| **GitHub Pages** | 저장소에 push → Settings → Pages → 브랜치 지정. `/<repo>/` 하위 경로로 배포돼도 모든 경로가 상대경로라 정상 동작 |

> 배포해도 **사용자 데이터는 서버에 저장되지 않습니다**(각 방문자의 브라우저에만 저장). 기본으로는 387만 원 예시 데이터만 들어 있습니다.

### 계산 단위테스트

계산 엔진(`src/calc.js`)은 Node로 검증 가능한 순수 함수입니다.

```bash
npm test                 # = node tests/calc.test.mjs
#   정상 시 "총 8개 테스트 통과 ✅" 출력
```

## 데이터 저장 (로컬 전용)

- 모든 데이터는 브라우저 **`localStorage`** 에 저장됩니다(키: `loan-app:v1:agreements`). 서버로 전송되지 않습니다.
- **내보내기**: 전체 데이터를 `loan-backup-YYYYMMDD.json` 파일로 다운로드.
- **가져오기**: JSON 파일 선택 → 스키마 검증 후 복원(불일치 시 거부).

## 제약 (반드시 준수)

- **개인·금융정보의 외부 전송 절대 금지.** 외부 서버·스크립트·폰트·이미지 URL·CDN으로 사용자 데이터를 보내지 않습니다. 모든 데이터는 기기 로컬(localStorage)에만 저장됩니다.
  - 단, 오프라인 지원을 위한 **서비스워커(`sw.js`)** 는 예외적으로 `fetch`를 사용하지만 **앱 자신의 정적 파일(동일 출처)** 만 캐시하며, 사용자 데이터는 다루지 않습니다.
- **빌드 단계 없음.** 프레임워크/번들러 없이 순수 HTML+CSS+Vanilla JS(ES Modules)만 사용.
- **금액**: 정수 ‘원’ 단위, `0 ~ 100,000,000,000원(1,000억)`. 부동소수점 누적오차 금지.
- **이율**: `0 ~ 20%`. 연 20% 초과 입력은 **저장 차단 + 경고**(이자제한법).
- **날짜**: `YYYY-MM-DD`(ISO). 주민등록번호는 **선택 입력 + 화면 마스킹**.
- 한글 라벨, 모바일 반응형, 키보드 입력 가능.

## 면책 · 주의

이 앱과 문서는 **세무·법률 자문이 아니며**, 일반 정보 제공·계산 보조 도구입니다. 차용증의 법적 효력, 증여세·이자소득 과세 여부 등은 **세무사·변호사 등 전문가와 개별 상담**이 필요합니다.

본문의 세제·이자율 수치(적정이자율 4.6%, 무이자 한도 약 2.17억, 최고이자율 20% 등)는 작성 기준일(2026-06-14) 시점이며 **법령·고시 개정으로 달라질 수 있습니다.** 데이터는 기기 내 로컬에만 저장되며, 백업 JSON 파일의 보관도 사용자 책임입니다.

> 전체 면책 문구는 `src/model.js`의 `DISCLAIMER`(및 `LEGAL_INFO`) export를 그대로 화면에 노출합니다.
